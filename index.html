Set report link in Settings

Special Day / Event Overrides

Flag an upcoming date (holiday, big game, promotion) so the forecast doesn't just use a normal weekday average for it. Applies automatically when that date becomes the next forecasted business day.

Import Hourly Sales

Import your hourly sales $ report the same way as Item Mix. Rows are summed into a daily total per Business Date and used to build a cases-per-$1,000 ratio for each product.

Projected Sales $ (for the $ Cross-Check)

Tell the forecast you expect a specific sales $ figure for an upcoming date (e.g. from a labor-scheduling projection). It'll blend a $-based case estimate in with the normal weekday average for that date, once enough historical days have both sales $ and case data.

New Item Proxy Forecasting

For a brand-new product with little or no sales history yet, borrow a ratio of another product's forecast until it has 5 consecutive days of its own data. It phases out automatically as real data builds up.

Backup & Restore

All data lives only in this browser. Export a backup regularly, especially before clearing a browser cache or switching devices.

Storage & Housekeeping

Old data is trimmed automatically each time the app opens: sales/case history older than ~13 months, layout history older than 6 months, and any Special Day or Projected Sales entry for a date that's already passed. This keeps browser storage from growing forever while still leaving enough history for year-over-year Special Day copies.

Data

Factory Reset

Erases everything on this device — forecast history, counts, layout history, freshness tracking, Special Days, proxies, sales data, settings, and the Director PIN (back to default 1234). Use this to start completely from zero, e.g. before real day-one use. This cannot be undone — export a backup first if there's anything worth keeping.
